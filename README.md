The Gaming Room Software Design

The client for this project was The Gaming Room, a company that developed the game Draw It or Lose It. The company wanted to expand its Android-based game to multiple operating systems, including Windows, Linux, Mac, and mobile devices. My task was to create a software design document that evaluated different operating platforms and recommended the best solution for scalability, performance, and security.

I believe I did particularly well in organizing the software design document and clearly explaining the advantages and disadvantages of different operating platforms. I also developed recommendations for operating systems, storage management, memory management, distributed systems, and security requirements.

Working through the design document helped me better understand the software development process before writing code. It allowed me to analyze requirements, evaluate alternatives, and think about how different components work together. This process will help me in future projects because good design can reduce errors and improve software quality.

If I could revise one part of my work, I would improve the evaluation section by providing more detailed comparisons of server-side hosting options and development tools. This would make the document even stronger and provide more insight into the costs and technical considerations of each platform.

I interpreted the user's needs by focusing on scalability, security, and cross-platform compatibility. Considering user needs is important because software should solve the client's problem while providing a positive experience for end users. Throughout this course, I used techniques such as requirement analysis, object-oriented design, and architectural evaluation to design the application. In the future, I plan to continue using these strategies to analyze and design similar software systems.
